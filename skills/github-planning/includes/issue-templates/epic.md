# Epic Issue Template

For STANDARD and COMPLEX tier tasks - parent issue linking sub-issues.

## Template

```markdown
## Overview

{{description}}

## Architecture

{{architecture}}

## Sub-Issues

{{#each sub_issues}}
- [ ] #{{this.number}} - {{this.title}}
{{/each}}

## Success Criteria

{{#each success_criteria}}
- {{this}}
{{/each}}

## Technical Decisions

{{#each decisions}}
### {{this.title}}
**Decision:** {{this.choice}}
**Rationale:** {{this.rationale}}
{{/each}}

## Dependencies

{{#if dependencies}}
{{#each dependencies}}
- {{this}}
{{/each}}
{{else}}
None identified.
{{/if}}

## Context

- **Complexity:** {{tier}} ({{score}}/15)
- **Estimated sub-tasks:** {{task_count}}
- **Created by:** /brainstorm
- **Date:** {{date}}

## Progress

- [ ] All sub-issues created
- [ ] Implementation started
- [ ] All sub-issues closed
- [ ] Final review complete

---
*Generated by fyrsmithlabs github-planning*
```

## Usage

```bash
gh issue create \
  --title "[EPIC] {{title}}" \
  --label "epic" \
  --body "<populated template>"
```

## Field Mapping

| Field | Source |
|-------|--------|
| `description` | Brainstorm design summary |
| `architecture` | Design architecture section |
| `sub_issues` | Populated after sub-issue creation |
| `success_criteria` | From design validation |
| `decisions` | Key decisions from brainstorm |
| `dependencies` | Identified blockers/prereqs |
| `tier` | STANDARD or COMPLEX |
| `score` | Complexity assessment total |
| `task_count` | Number of sub-issues |
| `date` | Current date ISO format |

## Sub-Issue Linking

After creating sub-issues, update epic body:

```bash
# Get epic body, update sub-issues section, then:
gh issue edit <epic-number> --body "<updated body>"
```
